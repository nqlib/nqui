# Impeccable Design Context — nqui

Design guidance for AI-assisted UI work on **@nqlib/nqui** library surfaces: docs, marketing, demo app, and package branding.

---

## Design Context

### Users

**Who:** Developers and AI agents evaluating, installing, and implementing nqui components.

**Context:** They work in IDEs, read docs quickly, compare nqui to shadcn/Radix patterns, and need confidence that components are production-ready. Sessions are task-focused — install, pick a component, copy patterns, verify accessibility and density.

**Job to be done:** Understand what nqui offers, trust its design opinions, and ship consistent UI faster without fighting the system.

### Brand Personality

**Voice:** Calm · Precise · Trustworthy

**Tone:** Quietly opinionated. No hype. Show craft through spacing, hierarchy, and restraint — not decoration. Copy is direct and technical where it helps; never salesy.

**Emotional goals:** Confidence (this kit won't surprise me), clarity (I know what to use when), calm (dense information without visual noise).

### Aesthetic Direction

**Visual tone:** Refined utilitarian — warm paper surfaces, precise type hierarchy, subtle depth. Industrial clarity with human warmth (not cold gray SaaS, not playful toy UI).

**Theme:** Light-first with warm paper neutrals (hue ~95). Dark mode exists for consumer apps but library/marketing surfaces default to light.

**References (implicit in current system):**
- OKLCH token scales, 2+1 elevation model, unified control heights (h-6 / h-7 / h-8)
- Warm `--background` over cool gray; blue primary (hue 240) used sparingly for actions
- Dense, information-forward layouts — docs and demos, not billboard marketing

**Anti-references — explicitly avoid:**
- Generic AI slop: purple/cyan gradients, neon-on-dark, glassmorphism, gradient text, side-stripe callouts
- Bootstrap/Material admin templates: identical card grids, icon-above-heading patterns, timid gray-on-gray
- Flashy startup marketing: hero metrics, decorative sparklines, bounce animations, modal-heavy flows

### Design Principles

1. **Precision over spectacle** — Hierarchy from spacing, type weight, and surface alternation; not gradients, glow, or motion for its own sake.
2. **Show the system, don't decorate it** — Demos and docs teach patterns. Every visual choice should reinforce how consumers should compose nqui.
3. **Warm restraint** — Tint neutrals toward brand hue; keep accents rare (60-30-10). Primary blue is for actions and selection, not wallpaper.
4. **Developer trust** — Accessible contrast, predictable focus states, no tricks that break when copied into a real app.
5. **Distinctive, not derivative** — Avoid reflex fonts (Inter monoculture on marketing) and template layouts; nqui should feel designed, not generated.

## Technical Baseline (from codebase)

| Area | Current choice |
|------|----------------|
| Stack | React 18/19, Tailwind CSS v4, Radix UI, TypeScript |
| Color | OKLCH scales in `src/styles/colors.css`; warm paper light, subdued dark |
| Typography | Inter variable (library default) — consider distinctive display pairing for marketing only |
| Elevation | 2 inline surfaces (`background`, `muted/40`) + 1 elevated (`popover` + shadow) |
| Motion | Restrained; see `src/styles/motion.css` and nqui motion philosophy |
| Components | shadcn-compatible patterns; see `docs/nqui-skills/` |

When designing **library surfaces**, stay aligned with shipped tokens unless intentionally exploring a marketing-only direction (document any divergence).
